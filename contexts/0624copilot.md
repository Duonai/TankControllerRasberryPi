# 2026-06-24 Copilot 작업 맥락

## 작업 개요
- `control/player1_tracks.py`, `control/player2_turret.py` 중심으로 탱크 제어 입력 로직을 조정.
- 주요 초점은 2P 포탑 제어의 좌우/상하/발사 역할 정리, 손/팔 기준 안정화, 시각 가이드 추가.
- 성능 최적화 이후 어긋난 사용자 체감 기준을 다시 맞추는 작업이 이어졌음.

## 오늘 반영된 핵심 변경
### 1. 2P 제어 역할 재정리
- 사용자 체감 기준으로 다음 역할이 되도록 `control/player2_turret.py`를 조정함.
  - 왼팔: 포신 상하(`pitch`)
  - 오른팔: 포탑 좌우(`yaw`) + 발사(`grip`)
- `--flip` 미리보기 사용 시 체감 좌우와 MediaPipe 인덱스 좌우가 달라 보이는 문제를 반영해 실제 동작 기준으로 수정.

### 2. 손 좌우 매칭 안정화
- `Hands`의 handedness 결과나 화면 x정렬에만 의존하지 않고, `Pose`의 좌우 손목 위치와의 거리 기준으로 손을 매칭하도록 변경.
- 이 수정으로 pitch 가이드선이 반대편 손으로 점프하는 현상을 줄이도록 조정.

### 3. 포신 상하 기준선 표시 조정
- `pitch` 기준선이 몸통 쪽에 떠 있던 문제를 수정.
- 현재는 제어에 쓰는 손목 x 위치를 기준으로 기준선이 따라가며, 손목이 잠시 누락되면 어깨 위치를 fallback으로 사용.

### 4. 포탑 좌우 각도 기준 재설정
- 전완 각도는 `위 방향 = 0도` 기준으로 계산하도록 변경.
- 계산식은 `atan2(dx, -dy)`를 사용.
- 따라서 팔을 위로 들면 `0도` 부근, 좌우로 기울일수록 signed angle이 증가/감소하는 방식임.

### 5. yaw idle 구간 확대
- `--yaw-deadzone-deg` 기본값을 `12`도에서 `18`도로 확대.
- 전완이 수직 근처에서 조금 흔들릴 때 `LEFT/RIGHT`로 너무 빨리 넘어가지 않도록 조정.

### 6. 팔꿈치 기준 yaw 가이드 추가
- 오른팔 팔꿈치 위치를 중심으로 원형 yaw 가이드를 그림.
- 표시 항목:
  - 위쪽 기준선
  - `STOP +/- deadzone` 경계선
  - `STOP +/-18 deg` 텍스트
- 사용자가 어느 정도 기울여야 stop 구간을 벗어나는지 바로 볼 수 있게 함.

### 7. yaw 가이드 색상 분리
- yaw 가이드 색이 상체 스켈레톤 강조색과 비슷해 잘 안 보이던 문제를 수정.
- 현재는 가이드 원/기준선과 경계선을 별도 색상으로 표시하여 스켈레톤과 구분되도록 함.

## 현재 기준 동작 정리
- `control/player2_turret.py`
  - 왼팔 상하: 포신 상하 조절
  - 오른팔 전완 기울기: 포탑 좌우 회전
  - 오른손 `open -> grip`: 발사 트리거
- yaw 기준:
  - 위 방향 `0도`
  - stop deadzone 기본 `±18도`

## 검증 메모
- 반복적으로 아래 명령으로 문법 검증 수행:
  - `python -m py_compile control/player2_turret.py`
- 실행 테스트는 주로 아래 명령 사용:
  - `python control/player2_turret.py --flip --camera-id 2 --model-complexity 0`
- 실행 후 종료 코드 `130`은 사용자가 직접 종료한 것으로 간주.
